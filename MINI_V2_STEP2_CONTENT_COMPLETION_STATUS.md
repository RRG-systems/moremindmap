## KEY LEARNINGS

1. **Fire-and-forget doesn't work** in Vercel serverless - Use poll-driven execution
2. **State machine needs explicit routing** - Don't assume stage names match function names
3. **Default vs named exports matter** - Dynamic imports need `.default` for default exports
4. **Redis serialization preserves types** - Arrays stay arrays, proved with test
5. **Field names must match exactly** - Generic dimension_N doesn't match template's vector/fidelity/etc.
6. **Boundary enforcement > caller assumptions** - Normalize at function entry, don't trust callers
7. **Trace logging is essential** - stage_trace array revealed the switch statement bug immediately
